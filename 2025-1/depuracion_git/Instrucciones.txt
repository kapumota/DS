## Flujo completo del proceso

1. **Compilación:**  
   El script utiliza `gcc` para compilar `auth.c` y `test_driver.c` en un ejecutable llamado `test_auth`.  
   - Si hay errores, se redirige la salida a `compile_error.log`.

2. **Ejecución:**  
   El ejecutable `test_auth` se ejecuta y su salida se redirige a `test_output.txt`.

3. **Validación:**  
   Se utiliza `diff` para comparar `test_output.txt` y `expected_output.txt`.  
   - Si coinciden, se considera que la prueba pasó (salida 0).  
   - Si difieren, se muestra la diferencia y el script retorna un error (salida 1).

Este conjunto de archivos y la forma en que interactúan permite automatizar las pruebas de la función `autenticarUsuario()` y es especialmente útil en un proceso de `git bisect` para determinar en qué commit se introdujo un fallo.
